# 프로젝트 상세 설계서: 다중 기사 합성 및 사용자 의도 파악 (Multi-Source Synthesis & Intent Discovery)

## 1. 개요
본 프로젝트는 '기사 분석 에이전트'를 사용자의 특정 분석 의도에 따라 다량의 소스를 정밀하게 합성하는 '서비스형 인사이트 엔진'으로 고도화하는 것을 목표로 합니다. 핵심 변화는 단일 기사 분석에서 다중 소스 합성으로의 전환과, 사용자의 세부 목표를 추적하는 인텐트 디스커버리 도입입니다.

## 2. 목표
- **사용자 의도 파악 (Intent Awareness)**: 사용자의 구체적인 분석 목표(예: 재무적 영향, 리스크 평가)를 추출하고 이를 분석 전체 과정에 반영합니다.
- **다중 소스 합성 (Multi-Source Synthesis)**: 3~5개의 뉴스 소스를 통합하여 균형 잡힌 종합 시각을 제공합니다.
- **견고성 및 성능 (Robustness)**: 고성능 데이터 수집 및 효율적인 리소스 관리 시스템을 구현합니다.

## 3. 아키텍처 및 구성 요소

### 3.1 Supervisor (분류 노드 강화)
- **확장된 책임**: `input_type` 분류뿐만 아니라, 사용자 입력에서 구체적인 '분석 목표(`analysis_objective`)'를 추출합니다.
- **출력**: `input_type`과 `analysis_objective`가 업데이트된 `AgentState`.

### 3.2 Collector (수집 노드)
- **로직 변경**: 
    - Tavily 검색 결과 중 상위 5개를 확보합니다.
    - `asyncio`를 사용하여 여러 기사 본문을 동시에 파싱합니다.
    - `analysis_objective`와의 관련성에 따라 기사를 필터링하고 우선순위를 결정합니다.
- **상태 업데이트**: 정제된 기사 데이터를 `article_list` 필드에 저장합니다.

### 3.3 Validator (검증 노드)
- **로직 변경**: 
    - 최소 2개 이상의 고품질 기사가 추출되었는지 확인합니다.
    - 수집된 컨텍스트가 `analysis_objective`를 달성하기에 충분한지 검토합니다.

### 3.4 Analyst (분석 노드)
- **다중 소스 프롬프팅**:
    - **합의점(Consensus)**: 모든 소스에서 공통적으로 언급되는 핵심 사실 식별.
    - **상충점(Contradictions)**: 서로 다른 관점이나 엇갈리는 수치 대조.
    - **전략적 인사이트**: `analysis_objective`에 맞춤화된 심층 통찰 제공.

## 4. 데이터 모델 (AgentState)
`graphs/state.py`의 다음 필드들이 추가되거나 수정됩니다:
- `article_list`: `list[dict[str, str]]` - 여러 기사의 메타데이터 및 본문 저장.
- `analysis_objective`: `str` - 사용자 입력에서 추출된 구체적인 분석 목표.
- `analysis_result`: 종합 합성 리포트를 담도록 구조 고도화.

## 5. 도구(Tools) 고도화
- **비동기 스크래퍼**: `article_parser_tool`이 배치 처리를 지원하게 하거나 비동기 래퍼를 구현합니다.
- **도메인 필터**: SSRF 방지를 위한 보안 체크 및 저품질 도메인 필터링 로직 추가.

## 6. 에러 처리 및 예외 상황
- **의도 불분명**: 사용자의 목표가 모호할 경우 Supervisor가 역질문을 통해 구체화를 요청합니다.
- **수집 실패**: 품질 기준을 만족하는 기사가 없을 경우, 검색 스니펫만을 활용한 제한적 요약을 제공하고 경고를 표시합니다.

## 7. 성공 기준
- 에이전트가 3개 이상의 소스에서 발생하는 상충 관계를 성공적으로 식별하고 해결함.
- 출력된 리포트가 사용자가 요청한 인사이트 카테고리와 정확히 일치함.
- 병렬 처리를 통해 전체 프로세스의 지연 시간을 최소화함.
