# Spec: Multi-Article Synthesis Agent - Premium Tabbed UI

본 문서는 `article_analyst_agent`의 사용자 인터페이스를 탭(Tab) 기반의 프리미엄 디자인으로 전환하기 위한 상세 설계를 정의합니다.

## 1. 개요
현재의 단일 채팅 인터페이스 방식은 다량의 정보(검색 결과, 분석 리포트, 후속 대화)가 혼재되어 사용자의 가독성을 저해합니다. 이를 해결하기 위해 기능별로 독립된 탭을 제공하여 "정리된 프리미엄 서비스"의 경험을 제공합니다.

## 2. 레이아웃 설계

### 가. 메인 탭 구조 (`st.tabs`)
1.  **💬 AI 대화 (Chat)**: 사용자와의 질의응답이 이루어지는 기본 캔버스입니다.
2.  **📌 수집된 소스 (Sources)**: 검색 단계에서 확보된 기사 원문 링크와 스니펫(미리보기)을 카드로 보여주는 라이브러리입니다.
3.  **📊 심층 리포트 (Analysis)**: 최종적으로 생성된 합성 분석 리포트를 전문 문서 형태로 보여주는 뷰어입니다.

### 나. 사이드바 (Sidebar)
- 세션 관리(초기화), 모델 정보, 현재 에이전트 워크플로우 상태를 상시 표시합니다.

## 3. 상세 컴포넌트 디자인

### 1) Tab 1: AI 대화
- **실시간 상태바**: `st.status`를 활용하여 `CLASSIFIER` -> `AGENT` -> `VALIDATOR` -> `ANALYST` 과정을 직관적으로 표시합니다.
- **메시지 버블**: 사용자(User)와 어시스턴트(Assistant)의 대화를 고대비 스타일로 구분합니다.

### 2) Tab 2: 수집된 소스 (Article Cards)
- **카드 구성**: `container`를 활용하여 개별 기사를 박스 형태로 디자인합니다.
    - 상단: `[언론사]` 기사 제목 (클릭 시 새 탭 이동)
    - 중단: 검색 엔진이 제공한 스니펫 (2~3줄)
    - 하단: 수집 성공/실패 여부를 나타내는 배지
- **데이터 소스**: `AgentState`의 `search_results` 및 `article_list` 필드를 활용합니다.

### 3) Tab 3: 심층 리포트
- **문서 스타일링**: 리포트 전용 CSS 클래스를 적용하여 폰트 크기, 줄 간격, 헤더 디자인을 최적화합니다.
- **다운로드 인터페이스**: 상단 또는 하단에 눈에 띄는 다운로드 버튼(`st.download_button`)을 배치합니다.

## 4. 기술적 고려 사항
- **상태 동기화**: 분석이 완료되면(`analysis_done`) 자동으로 '심층 리포트' 탭에 결과가 반영되도록 세션 상태(`st.session_state`)를 연동합니다.
- **UI 반응성**: Streamlit의 기본 리런(Rerun) 메커니즘에서도 수집된 기사 목록이 휘발되지 않도록 `article_list`를 세션 상태와 철저히 동기화합니다.
- **CSS 고도화**: `Pretendard` 폰트 임베딩 및 카드에 마우스 오버 시 그림자 효과 등을 추가하여 프리미엄 감성을 구현합니다.

## 5. 성공 기준
- 사용자가 검색 결과와 최종 리포트 사이를 탭 전환만으로 손쉽게 오갈 수 있어야 함.
- 분석 로딩 시 어떤 정보를 수집 중인지 '수집된 소스' 탭에서 실시간으로 확인할 수 있어야 함.
- 모바일 및 데스크톱 환경 모두에서 깨지지 않는 반응형 레이아웃 유지.
